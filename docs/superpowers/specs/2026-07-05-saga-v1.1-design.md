# Saga v1.1 - UI overhaul + summary UX features

Dato: 2026-07-05.
Status: godkjent design (valgt fra anbefalte alternativer 2026-07-05), venter spec-review + implementasjonsplan.
Bygger på: Saga v1 (live på saga.kongebra.no, prod-only). Backend `apps/saga-api`, frontend `apps/saga-web` (TanStack Start).

## Mål

Løfte Saga fra funksjonell-men-rå til polert, og legge til fire summary-UX-features.
Rekkefølge er bevisst: **UI-fundament (tailwind + shadcn/ui) FØRST**, så bygges de andre featurene på det nye komponentsystemet - ikke i inline-styles som så kastes.

## Beslutninger

| Tema | Beslutning | Hvorfor |
|---|---|---|
| Rekkefølge | UI-fundament først, så features 1-3 | Feature 1-3 er alle UI-tillegg; å bygge dem i inline-styles og så restyle = dobbeltarbeid |
| Video-visning | Thumbnail + innebygd YouTube-embed (spiller inline på klikk) | Rikest; metadata via YouTube oEmbed = null backend-endring |
| Metadata-kilde | YouTube oEmbed (client-side): tittel, kanal, thumbnail | CORS-åpent, ingen API-nøkkel, ingen backend/DB-endring |
| Oversett | Toggle EN/NO, cache begge | Sammendrag genereres på engelsk (ny default); knapp oversetter én gang via Ollama, lagres; toggle uten re-kjøring |
| Tema | Light + dark m/ toggle | shadcn støtter begge gratis; standard for verktøy-UI |
| shadcn-omfang | Kjerne: button, card, input, select, badge, skeleton + prose-typografi + lucide-ikoner | Polert uten scope-eksplosjon |
| Default-språk | Engelsk (endres fra "no") | Brukers valg; norsk via oversett-knapp ved behov |

## Arkitektur (to faser, én spec)

### Fase A - UI-fundament (saga-web, ingen backend)

Tailwind v4 (`@tailwindcss/vite`-plugin) + shadcn/ui-komponenter, erstatter alle inline-styles.

- `vite.config.ts`: legg til `@tailwindcss/vite`-plugin.
- `src/styles.css`: tailwind + shadcn design-tokens (CSS-variabler for light + dark via `.dark`-klasse på `<html>`), importeres i `__root.tsx`.
- `src/lib/utils.ts`: `cn()` (clsx + tailwind-merge) - shadcn-standard.
- `src/components/ui/`: shadcn-komponenter (button, card, input, select, badge, skeleton) - genereres/kopieres, eies i repo (shadcn er copy-in, ikke npm-runtime-dep).
- Nye deps: `tailwindcss` + `@tailwindcss/vite` (v4), `@tailwindcss/typography` (prose), `clsx`, `tailwind-merge`, `class-variance-authority`, `lucide-react`, `@radix-ui/react-slot` + de radix-primitives shadcn-komponentene trenger (select bruker `@radix-ui/react-select`). Alle pinnes.
- Tema: liten egen theme-provider (localStorage + `.dark`-klasse på `document.documentElement`, respekterer `prefers-color-scheme` ved førstegangs-last). Ikke `next-themes` (Next-spesifikk). Toggle-knapp (sol/måne lucide-ikon) i `Shell`-headeren.
- Restyle: `ui.tsx` (Shell, StatusPill → shadcn Card/Badge), `routes/index.tsx` (form → Input/Select/Button, jobbliste → Card-rader), `routes/jobs.$id.tsx` (Card-layout), `markdown.tsx` (wrap i `prose dark:prose-invert`).
- SSR/hydrering: tema-klassen settes av et lite inline-script i `<head>` (`__root.tsx`) FØR hydrering, så det ikke blinker (FOUC). Markdown fortsatt client-only (marked+DOMPurify).

### Fase B - features (bygges på fase A)

**1. ETA på chunking (frontend-only)**
- I `jobs.$id.tsx` SSE-handleren: parse `chunk i/n` fra `ev.detail`, tidsstemple (`Date.now()`), regn snitt-per-chunk og projiser gjenstående: `(n - i) * (elapsed / chunksDone)`.
- Vis i progress-linja: `chunk 3/12 - ~2m igjen`. Nullstill ved (re)åpning av stream. Grovt estimat, merket som sådan.

**2. Video-metadata + thumbnail + embed (frontend-only, oEmbed)**
- `src/lib/youtube.ts`: `videoId(url): string | null` (parse watch?v=, youtu.be/, embed/).
- Detaljside: hent `https://www.youtube.com/oembed?url=<url>&format=json` (client fetch, CORS-ok) → `title`, `author_name` (kanal), `thumbnail_url`. Vis thumbnail-kort med tittel + kanal.
- Embed: klikk thumbnail → bytt til `<iframe src="https://www.youtube-nocookie.com/embed/<id>">` (youtube-nocookie = mindre tracking) inline. `loading="lazy"`.
- Ingen backend-endring, ingen DB-felt. Faller pent tilbake (viser bare tittel fra sammendraget) hvis oEmbed feiler / ikke-YouTube-URL.

**3. Default engelsk + oversett-til-norsk (backend + frontend)**
- Backend default-språk: `ytsummary` endrer default fra `"no"` → `"en"`. Frontend form-default: `"en"`.
- Migrasjon `0002_translated.sql`: `ALTER TABLE jobs ADD COLUMN translated_markdown text; ADD COLUMN translated_lang text;`.
- `summarize.TranslatePrompt(targetLang, markdown) string` - instruer LLM: oversett Markdown-en til målspråk, behold Markdown-struktur.
- Endpoint `POST /api/jobs/{id}/translate` body `{"lang":"no"}`: krever `status=done` + `result_markdown` (ellers 409); kjører `result_markdown` gjennom `llm.Chat` (samme n=1-chokepoint som worker - semaforen serialiserer), lagrer `translated_markdown`+`translated_lang`, returnerer `{"translated_markdown": "..."}`. Non-streaming (et kort sammendrag oversettes på ~10-20s; skeleton vises imens). `# ponytail:` non-streaming; upgrade-path = SSE-token-stream hvis det oppleves tregt.
- `GET /api/jobs/{id}` inkluderer `translated_markdown` + `translated_lang`.
- Frontend: på done-jobb, en EN/NO-toggle. NO første gang → POST translate (skeleton) → vis. Cachet: hvis `translated_markdown` finnes, toggle uten kall.

## Dataflyt (oversett)

1. Bruker på done-jobb (engelsk sammendrag) klikker "Norsk".
2. Hvis `translated_markdown` allerede satt → vis den (ingen kall).
3. Ellers: `POST /api/jobs/{id}/translate {lang:"no"}` → server kjører LLM → lagrer + returnerer → frontend viser, skeleton imens.
4. Toggle tilbake til "English" → vis `result_markdown`.

## Feilhåndtering

- oEmbed-feil / ikke-YouTube-URL → fall tilbake til tittel-fra-markdown, ingen thumbnail/embed (ingen krasj).
- Translate på ikke-done-jobb → 409. LLM-feil under translate → 500 + feilmelding i UI (toggle tilbake til EN).
- ETA når < 1 chunk fullført → vis ingen ETA ennå (unngå divisjon på null).
- Tema-toggle: default til OS-preferanse; localStorage overstyrer.

## Testing

- saga-api: `TranslatePrompt` unit-test (målspråk + input i prompten); translate-endpoint mot ekte PG (translate en done-jobb → `translated_markdown` lagret; 409 på ikke-done). Migrasjon idempotent.
- saga-web: `videoId()` unit-test (de tre URL-formene + ikke-YouTube → null); ETA-beregning ren funksjon → unit-test. Ellers typecheck + build + clean-install (samme gate som v1). Playwright E2E for oversett-toggle + embed.

## Deploy

Prod-only CI (allerede på plass): merge → build → deploy-prod-gate → ArgoCD. Migrasjon `0002` kjøres av saga-api ved oppstart (embedded migrations). Ingen gitops-manifest-endring nødvendig (samme image/deployment).

## Utenfor scope v1.1

- Streaming av oversettelse (non-streaming nå; SSE-upgrade dokumentert).
- Varighet-basert ETA (bruker chunk-timing, ikke video-varighet - unngår backend-metadata).
- Andre målspråk enn norsk (toggle er EN↔NO; generaliser hvis behov).
- Backend-lagret video-metadata (oEmbed client-side dekker behovet).
